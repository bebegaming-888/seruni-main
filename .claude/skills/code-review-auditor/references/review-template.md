## Scoring Guide

### Correctness (0-20)
- **18-20:** No logic errors, proper null checks, correct types, edge cases handled
- **12-17:** Minor logic issues, some edge cases missed
- **6-11:** Logic errors present, significant edge cases not handled
- **0-5:** Major bugs present or missing fundamental checks

### Security (0-15)
- **13-15:** No security issues, proper validation, auth, secrets handled correctly
- **9-12:** Minor security gaps, non-critical
- **4-8:** Significant security concerns (missing input validation, weak auth)
- **0-3:** Critical security issues (SQL injection, auth bypass, hardcoded secrets)

### Performance (0-10)
- **9-10:** No performance issues, proper pagination, no N+1
- **6-8:** Minor performance concerns
- **3-5:** N+1 or missing pagination present
- **0-2:** Critical performance issues (unbounded loading, blocking sync ops)

### Maintainability (0-10)
- **9-10:** Clean code, good naming, no duplication
- **6-8:** Minor issues (some magic numbers, slightly long functions)
- **3-5:** Significant maintainability concerns
- **0-2:** Code is hard to maintain (duplication, bad naming, 500-line functions)

### Test Coverage (0-10)
- **9-10:** Comprehensive tests, edge cases covered
- **6-8:** Core paths covered, some edge cases missing
- **3-5:** Basic tests, significant gaps
- **0-2:** No tests or tests don't exercise core logic

### API Contract (0-5)
- **5:** No breaking changes, proper versioning
- **3-4:** Minor API changes, documented
- **1-2:** Breaking changes, no version bump
- **0:** Breaking changes with no mitigation
